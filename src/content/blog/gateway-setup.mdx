## Telegram Setup

Telegram is the most popular gateway platform. Here's how to set it up:

### 1. Create a Bot

1. Open Telegram and message [@BotFather](https://t.me/botfather)
2. Send `/newbot`
3. Choose a name (e.g., `My Hermes Agent`)
4. Choose a username (e.g., `my_hermes_bot`)
5. BotFather gives you a **token** â€” save it

### 2. Configure Hermes

```bash
hermes gateway setup
```

Select Telegram and paste the token when prompted. Or configure manually:

```yaml
# ~/.hermes/config.yaml
gateway:
  platforms:
    telegram:
      enabled: true
      bot_token: "1234567890:ABCdefGHIjklmNOPqrstUVwxyz"
```

### 3. Start and Chat

```bash
hermes gateway install
hermes gateway start
```

Now message your bot on Telegram. The agent has full tool access â€” you can ask it to search the web, run commands, analyze images, or manage files, all from your phone.

**Telegram-specific features:**

- `/sethome` â€” set current chat as the "home" channel
- `/topic` â€” enable DM topic sessions
- Voice messages are auto-transcribed (requires STT setup)
- `/platforms` â€” check connection status

### 4. Authorize Users (DM Commands)

By default, anyone who knows your bot can message it. Control access:

```bash
hermes pairing list           # Show authorized users
hermes pairing approve "@user"  # Approve a user
hermes pairing revoke "@user"   # Revoke access
```

## Discord Setup

### 1. Create a Bot Application

1. Go to [Discord Developer Portal](https://discord.com/developers/applications)
2. New Application â†’ name it
3. Bot â†’ Add Bot â†’ copy the token
4. **IMPORTANT:** Enable **Message Content Intent** under Privileged Gateway Intents
5. OAuth2 â†’ URL Generator â†’ scopes: `bot` + `applications.commands` â†’ permissions: `Send Messages`, `Read Message History` â†’ open the generated URL to invite the bot

### 2. Configure

```yaml
# ~/.hermes/config.yaml
gateway:
  platforms:
    discord:
      enabled: true
      bot_token: "your_discord_bot_token"
```

### 3. Start

```bash
hermes gateway restart
```

**Discord-specific notes:**

- The bot responds in DMs and any channel it has permission to read
- Without **Message Content Intent** enabled, the bot only sees DMs, not channel messages
- Slash commands aren't supported yet â€” Hermes listens to plain messages

## Slack Setup

### 1. Create a Slack App

1. Go to [api.slack.com/apps](https://api.slack.com/apps)
2. Create New App â†’ From Manifest
3. Add these **Bot Token Scopes**: `chat:write`, `im:history`, `im:read`, `channels:history`, `channels:read`
4. Subscribe to `message.channels` and `message.im` events
5. Install to Workspace â†’ copy the **Bot User OAuth Token**

### 2. Configure

```yaml
# ~/.hermes/config.yaml
gateway:
  platforms:
    slack:
      enabled: true
      bot_token: "xoxb-..."
```

### 3. Start

```bash
hermes gateway restart
```

**Slack-specific note:** Without subscribing to `message.channels`, the bot only responds in DMs. Subscribe to that event to make it work in public channels too.

## Gateway Management

Once the gateway is running, you can manage it with:

```bash
hermes gateway status          # Running or stopped?
hermes gateway logs            # Show recent logs
hermes gateway restart         # Full restart
hermes gateway stop            # Stop the service
```

Check gateway logs for issues:

```bash
grep -i "failed to send\|error" ~/.hermes/logs/gateway.log | tail -20
```

## Common Pitfalls

### Gateway dies on SSH logout

```
sudo loginctl enable-linger $USER
```

This keeps user services alive after you disconnect.

### Gateway dies on WSL2 close

WSL2 needs `systemd=true` in `/etc/wsl.conf` for systemd services to work. Without it, the gateway falls back to `nohup` (dies when terminal closes).

### Discord bot is silent

Double-check **Message Content Intent** is enabled in Discord Developer Portal â†’ Bot â†’ Privileged Gateway Intents.

### Slack bot only works in DMs

Subscribe to `message.channels` event in your Slack App configuration.

### Gateway crash loop

```bash
systemctl --user reset-failed hermes-gateway
hermes gateway start
```
